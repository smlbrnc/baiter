## Bulunan bug ve düzeltme

### Sorun

Önceki simülatör implementasyonunda:
```python
if locked:
    continue  # tüm aksiyonlar atlanır (signal_flip dahil)
```

Lock devresine girdikten sonra **signal_flip de bloke olduğu için** bot yön değiştiremedi.
Bu marketde t=10'da lock'a girdi → t=74'te score sıçramasına rağmen flip yapamadı
→ sadece 3 trade, -$7 kayıp.

### Düzeltme

```python
if signal_flip_triggered:
    # ... flip et ...
    continue  # her zaman, lock'a aldırmaz

if locked:
    continue  # diğer aksiyonları atla
```

Yön değişikliği lock'tan **mantıken önce** kontrol edilmeli — eğer piyasa ters döndüyse
"kâr garantili" varsayımı çürür, lock geçersiz olur.
